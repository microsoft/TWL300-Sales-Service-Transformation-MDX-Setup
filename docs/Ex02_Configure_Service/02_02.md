## Task 02: Create and manage cases

### Introduction

Cases are the core work items in Dynamics 365 Customer Service. In this task, you will create a case, link it to an existing account, and resolve it using the case resolution process.

### Description

In this task, you will create a service case for an existing account, assign it to a queue, and resolve the case.

### Success criteria

- A case record is created and linked to an account.
- The case is assigned to the correct queue.
- The case is resolved and the resolution is recorded.

### Key steps:

1. Create a case.

    <details markdown="block">
    <summary>Expand here for detailed steps</summary>

    1. In Customer Service Hub, select **Cases** from the left navigation.
    1. Select **+ New Case**.
    1. Enter the following values:

        | Field | Value |
        |:------|:------|
        | Case Title | `Unable to access portal` |
        | Customer | `Contoso Ltd.` |
        | Priority | `High` |

    1. Select **Save**.

    </details>

1. Assign the case to a queue.

    <details markdown="block">
    <summary>Expand here for detailed steps</summary>

    1. On the case record, select **Add to Queue** from the command bar.
    1. Select **General Support Queue**.
    1. Select **Add**.

    </details>

1. Resolve the case.

    <details markdown="block">
    <summary>Expand here for detailed steps</summary>

    1. On the case record, select **Resolve Case** from the command bar.
    1. Enter a **Resolution** description: `Provided the user with portal reset instructions.`
    1. Select **Resolve**.

    </details>
