## Task 01: Set up queues and routing rules

### Introduction

Queues and routing rules direct incoming cases to the right teams and agents. In this task, you will create a service queue, define routing rules, and verify that cases are automatically assigned based on the configured criteria.

### Description

In this task, you will access the Customer Service Hub, create a queue for the support team, and configure a routing rule that assigns cases with a specific category to that queue.

### Success criteria

- A support queue is created and active.
- A routing rule is configured to route cases to the queue based on case category.
- A test case is routed correctly to the queue.

### Key steps:

1. Create a queue.

    <details markdown="block">
    <summary>Expand here for detailed steps</summary>

    1. Open **Customer Service Hub** from the app list.
    1. Select the settings gear icon and then **Advanced Settings**.
    1. Go to **Settings** > **Service Management** > **Queues**.
    1. Select **+ New**.
    1. Enter the following values:

        | Field | Value |
        |:------|:------|
        | Name | `General Support Queue` |
        | Type | `Public` |

    1. Select **Save**.

    </details>

1. Configure a routing rule.

    <details markdown="block">
    <summary>Expand here for detailed steps</summary>

    1. Go to **Settings** > **Service Management** > **Routing Rule Sets**.
    1. Select **+ New**.
    1. Enter the **Name** as `Default Routing Rules`.
    1. Select **Save**, then select **+ New Rule Item**.
    1. Set the **Case Category** condition and assign to **General Support Queue**.
    1. Select **Save and Close**.
    1. Activate the routing rule set by selecting **Activate**.

    </details>
